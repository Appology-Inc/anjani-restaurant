# Anjani Ecosystem Workspace Rules

## 1. Documentation & Code Quality (CRITICAL)
Whenever adding new features, editing existing files, or writing new code anywhere in this workspace, you MUST maintain the absolute highest standard of professional documentation:
- **File Headers**: Every file must have a top-level `@file` and `@description` JSDoc header.
- **Function/Component Documentation**: Every function, React component, or interface must be annotated with complete JSDoc/TSDoc blocks explaining inputs, outputs, and side-effects.
- **Inline Comments**: Complex logic, state transitions, API calls, and business logic must be accompanied by conversational, easy-to-read inline comments.
- The goal is to ensure the codebase remains 100% "Open-Source Ready" and easily readable by community developers at all times. Failure to document code is unacceptable.
